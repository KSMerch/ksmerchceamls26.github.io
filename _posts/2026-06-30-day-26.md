what_i_learned: |
  Today, we traveled to Washington, D.C. to attend the AWS Summit 2026. Throughout the day, I attended several breakout sessions that focused on cybersecurity, artificial intelligence, and cloud computing. The event
  gave me the opportunity to learn about how AI is being used in real-world industries and the challenges that come with integrating it into security and software development workflows. One of the sessions I attended 
  was "Outpace the Adversary: Cyber Defense at AI Speed" presented by Booz Allen. In this session, I learned how artificial intelligence is changing the cybersecurity landscape. AI is increasing the speed, volume, 
  and sophistication of cyberattacks, making it more difficult for human analysts to keep up with threats. The speakers discussed how malware can evolve rapidly and how attackers can use AI to automate and improve 
  their techniques. I also learned how organizations are beginning to use agentic AI defensively by embedding cybersecurity expertise, attack knowledge, and response strategies into AI systems that can operate at 
  machine speed. Another session I attended was "AI Agents Don't Bypass Your Policy Gates. Your Configuration Does." This presentation focused on security compliance and software development pipelines. I learned
  that many organizations have security checks in place, but those checks are not always properly enforced. The speakers explained the importance of moving from systems that simply scan for issues to systems that 
  actively enforce security policies. I also learned about concepts such as Policy-as-Code and how organizations can manage the risks associated with AI-generated code by implementing stronger automated controls and 
  compliance frameworks. The final session I attended was "AWS Security Agent: Get Started with AI-Powered Proactive AppSec." During this workshop, I learned how AI can be used to improve application security 
  throughout the software development lifecycle. The session demonstrated how AI agents can review system architectures for security concerns before code is even written and can also perform penetration testing to 
  identify vulnerabilities. I learned about automated remediation tools that can suggest or generate code fixes for security issues, helping developers address vulnerabilities earlier in the development process. 
  Overall, today showed me how AI is being applied beyond machine learning research and into areas such as cybersecurity, software engineering, and cloud infrastructure. It was interesting to see how organizations 
  are using AI not only to build new technologies but also to defend against emerging threats.
