what_i_learned: |
  Today did not go exactly as planned because the training data from yesterday was not saved when the Google Colab runtime disconnected. As a result, I had to restart
  the training process and spend time getting everything running again. Although this was frustrating, it reinforced the importance of properly saving results and 
  managing long-running machine learning experiments. While waiting for the model to train, I spent time learning about ROC curves, AUC, and false positive and 
  false negative rates. I learned that a ROC (Receiver Operating Characteristic) curve is used to visualize how well a model distinguishes between different 
  classes by comparing the true positive rate and false positive rate at different thresholds. I also learned that AUC (Area Under the Curve) provides a single 
  value that summarizes the model's overall ability to separate positive and negative cases, with higher values indicating better performance. Additionally, I learned 
  about false positive rates (FPR) and false negative rates (FNR). A false positive occurs when a model predicts a condition is present when it is not, while a 
  false negative occurs when the model fails to detect a condition that actually exists. Understanding these metrics helped me see how different types of prediction 
  errors can impact model performance, especially in medical applications where missing a diagnosis can have serious consequences. Overall, today helped me better 
  understand some of the evaluation methods that researchers use to assess machine learning models beyond basic accuracy measurements.
