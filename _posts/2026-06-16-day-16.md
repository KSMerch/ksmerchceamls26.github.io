what_i_learned: |
  Today, we started by looking over the agenda for the week and discussing some of the concepts that will be important for our project. One topic we focused on was the 
  difference between multi-label and multi-class classification. I learned that in a multi-class problem, each example can belong to only one category, while in a 
  multi-label problem, an example can belong to multiple categories at the same time. This is especially relevant to ECG data because a patient may have more than one 
  heart condition present in a single recording. We also watched videos that explained precision, recall, and F1 score. I learned that these metrics provide a more 
  detailed way to evaluate a machine learning model than accuracy alone. Precision measures how often positive predictions are correct, recall measures how many 
  actual positive cases are found, and the F1 score balances both metrics into a single value. Understanding these measures helped me see how researchers evaluate 
  whether a model is performing well, especially in situations where some conditions may be less common than others. Later in the day, we worked on modifying our 
  training code so that model files and results would be saved directly to Google Drive. Previously, we had to wait through long training sessions if files were lost 
  when a runtime disconnected. By saving directly to Google Drive, we can preserve our work and avoid unnecessary retraining. This change made the workflow more 
  efficient and showed me how important it is to think about practical aspects of managing machine learning experiments.
